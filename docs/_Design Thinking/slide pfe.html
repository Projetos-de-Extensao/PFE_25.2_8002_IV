Introdução: Contexto e Objetivo

  • Contexto: O gerenciamento do programa de monitorias é um processo manual, dificultando o processo seletivo, o controle de frequência e a análise de dados.
  • Objetivo: Desenvolver uma plataforma digital para gerenciar o processo seletivo de monitores e centralizar o acompanhamento das monitorias.
  • Público-Alvo: Monitores, Professores, Coordenadores e Alunos.

Requisitos Funcionais

O que o sistema fará?

  • Permitir que alunos se candidatem para vagas de monitoria.
  • Permitir que professores analisem e aprovem candidatos.
  • Disponibilizar ao coordenador um painel para cadastrar as aulas de monitoria.
  • Permitir que o monitor lance a presença dos alunos atendidos.
  • Apresentar dashboards para monitor e coordenador acompanharem o desempenho.
  • Exibir para os alunos a grade de aulas de monitoria do dia.

Requisitos Não Funcionais

Como o sistema será?

  • **Performance:** A interface deve carregar rapidamente e apresentar informações de forma clara.
  • **Responsividade (BS09):** Aplicação adaptável a desktops, tablets e smartphones.
  • **Identidade Visual (BS13):** Paleta de cores e tipografia consistentes.
  • **Feedback ao Usuário (BS14):** Incluir loaders e mensagens de erro/sucesso amigáveis.
  • **Usabilidade:** A navegação entre as diferentes seções deve ser simples e intuitiva para cada perfil.

Caso de Uso 1: Jornada do Aluno (Candidato)

Como um aluno se torna monitor?

  1. Acessa a plataforma e visualiza as aulas de monitoria que precisam de monitores.
  2. Clica em uma vaga de interesse para ver os detalhes.
  3. Submete sua candidatura para a vaga desejada.
  4. Acompanha o status de sua candidatura (pendente, aprovado, etc.) em seu painel.
  5. Uma vez aprovado, seu perfil muda para "Monitor".

Caso de Uso 2: Jornada do Monitor

Como a interface do monitor funciona?

  1. Acessa seu dashboard e visualiza sua agenda de aulas pré-definida pelo coordenador.
  2. Após uma aula, acessa a função "Lançar Presença" para registrar os alunos que participaram.
  3. Em seu painel de desenvolvimento, acompanha suas horas e o progresso dos alunos que monitora.
  4. Visualiza seu histórico de aulas e o feedback dos alunos.

Caso de Uso 3: Jornada do Professor

Como um professor seleciona os monitores?

  1. Acessa a plataforma e navega para a seção "Processo Seletivo".
  2. Visualiza a lista de alunos que se candidataram para monitoria em suas disciplinas.
  3. Analisa o perfil de cada candidato (histórico, notas, etc.).
  4. Aprova ou recusa os candidatos. Os alunos aprovados tornam-se monitores no sistema.

Caso de Uso 4: Jornada do Coordenador

Que informações o dashboard do coordenador deve exibir?

  1. **Cadastro:** Define os horários e disciplinas das aulas de monitoria na plataforma.
  2. **Visão Geral:** Gráficos com o número de monitorias (semana/mês) e alunos atendidos.
  3. **Engajamento:** Lista das disciplinas com maior e menor procura.
  4. **Desempenho:** Tabela com monitores, horas registradas e média de avaliações.
